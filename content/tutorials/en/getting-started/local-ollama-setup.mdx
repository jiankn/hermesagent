## What You'll Get After This Tutorial

By the end of this guide, you'll have Hermes Agent running entirely on your local machine — **no API keys, no cloud dependency, no recurring costs**. Your conversations stay on your hardware, and you can use it completely offline.

> **Perfect for:** Privacy-conscious users, developers in regions with API restrictions, students who don't want to pay for API access, or anyone who wants to tinker with open-source models.

## Prerequisites

- **Hermes Agent installed** — If not, follow our [installation guide](/en/tutorials/getting-started/installation) first
- **Hardware minimum:** 8GB RAM for 7B models, 16GB for 13B models, 32GB+ for 70B models
- **Disk space:** At least 5GB free (models range from 4GB to 40GB+)
- **GPU (optional but recommended):** Any NVIDIA GPU with 6GB+ VRAM dramatically speeds up inference

> **No GPU? No problem.** Ollama runs fine on CPU — it's just slower (expect 5-15 tokens/sec on a modern CPU vs 40-80 tokens/sec with a GPU).

## Step 1: Install Ollama

Ollama makes running local LLMs as easy as running Docker containers. One command:

### macOS / Linux

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

### Windows

Download the installer from [ollama.com/download](https://ollama.com/download) and run it. Ollama will run as a system service.

### Verify Installation

```bash
ollama --version
# Expected: ollama version 0.5.x
```

Ollama starts a local API server at `http://localhost:11434`. You can verify it's running:

```bash
curl http://localhost:11434
# Expected: Ollama is running
```

## Step 2: Choose and Download a Model

This is the fun part. Here's a comparison of popular models for agentic tasks:

| Model | Size | RAM Needed | Best For | Command |
|-------|------|-----------|---------|---------|
| **Llama 3.1 8B** | 4.7GB | 8GB | General tasks, fast responses | `ollama pull llama3.1` |
| **Llama 3.1 70B** | 40GB | 48GB+ | Complex reasoning, coding | `ollama pull llama3.1:70b` |
| **Mistral 7B** | 4.1GB | 8GB | Fast, good at following instructions | `ollama pull mistral` |
| **Mixtral 8x7B** | 26GB | 32GB | Balanced power/speed | `ollama pull mixtral` |
| **Qwen 2.5 7B** | 4.4GB | 8GB | Multilingual, strong at Chinese | `ollama pull qwen2.5` |
| **CodeLlama 7B** | 3.8GB | 8GB | Code generation | `ollama pull codellama` |
| **DeepSeek Coder V2** | 8.9GB | 16GB | Best open-source coder | `ollama pull deepseek-coder-v2` |

**Our recommendation for most users:**

```bash
# Best balance of quality and speed
ollama pull llama3.1

# Verify it downloaded
ollama list
```

Expected output:

```
NAME              ID           SIZE    MODIFIED
llama3.1:latest   a]b2c3d4e5   4.7 GB  2 minutes ago
```

### Quick Test

Make sure the model works before connecting it to Hermes:

```bash
ollama run llama3.1 "What is 2+2? Reply in one word."
# Expected: Four
```

Press `Ctrl+D` to exit the Ollama chat.

## Step 3: Connect Hermes Agent to Ollama

Now the magic — point Hermes at your local Ollama instance instead of a cloud API.

### Option A: Environment Variable (Quick)

```bash
export OPENAI_API_BASE=http://localhost:11434/v1
export OPENAI_API_KEY=ollama  # Ollama doesn't need a real key, but Hermes requires the field
export DEFAULT_MODEL=llama3.1
```

Then start Hermes:

```bash
hermes
```

### Option B: Configuration File (Permanent)

Edit your Hermes config file (`~/.hermes/config.yaml`):

```yaml
provider:
  type: openai-compatible
  base_url: http://localhost:11434/v1
  api_key: ollama  # placeholder — Ollama ignores this
  model: llama3.1

# Optimize for local models
settings:
  max_tokens: 2048        # Keep reasonable for 7B models
  temperature: 0.7
  context_window: 8192    # Llama 3.1 supports up to 128k, but 8k is efficient for most tasks
```

### Verify the Connection

```bash
hermes "Hello! Tell me what model you are."
```

You should see a response from Llama 3.1 (or whichever model you chose). The response will come from your local machine — no data leaves your computer.

```
🤖 Hello! I'm running as Llama 3.1 through Ollama on your local machine.
   How can I help you today?
```

## Step 4: Optimize Performance

### For CPU-Only Systems

If you're running without a GPU, these settings help:

```yaml
# In ~/.hermes/config.yaml
settings:
  max_tokens: 1024       # Shorter responses = faster
  context_window: 4096   # Smaller context = less memory
  num_ctx: 4096          # Match context_window
```

### For NVIDIA GPU Systems

Ollama automatically detects NVIDIA GPUs. Verify it's using your GPU:

```bash
ollama ps
```

You should see your model loaded with GPU layers:

```
NAME          ID          SIZE     PROCESSOR    UNTIL
llama3.1      a1b2c3d4    6.7 GB   100% GPU     4 minutes from now
```

### For Apple Silicon (M1/M2/M3/M4)

Great news — Ollama is optimized for Apple Silicon and uses the unified memory architecture. A MacBook with 16GB RAM can comfortably run 7B models at 30+ tokens/sec.

No special configuration needed. Just install and run.

## Running Multiple Models

One of Ollama's strengths is easy model switching. You can have multiple models downloaded and switch between them:

```bash
# Download a coding-focused model too
ollama pull deepseek-coder-v2

# Use it for coding tasks
hermes --model deepseek-coder-v2 "Write a Python script to parse CSV files"

# Switch back to general model
hermes --model llama3.1 "Summarize today's tasks"
```

Or set up model aliases in your config:

```yaml
# ~/.hermes/config.yaml
models:
  default: llama3.1
  coding: deepseek-coder-v2
  multilingual: qwen2.5
```

## Troubleshooting

### Ollama Not Running

**Q: Hermes says "Connection refused" or "Cannot connect to Ollama"?**

Make sure Ollama is running:

```bash
# Check if Ollama service is active
ollama serve

# Or on systemd-based Linux:
sudo systemctl status ollama
sudo systemctl start ollama
```

### Model Too Slow

**Q: Getting only 2-3 tokens per second?**

A: You're likely running a model that's too large for your hardware. Try:

1. Switch to a smaller model: `ollama pull mistral` (4.1GB, very fast)
2. Reduce context window in config to `2048`
3. Check if another program is using your GPU: `nvidia-smi`

### Out of Memory

**Q: Getting "out of memory" errors?**

A: The model doesn't fit in your RAM. Solutions:

```bash
# Use a quantized (smaller) version
ollama pull llama3.1:7b-q4_0    # 4-bit quantization, ~3.5GB

# Or use a smaller model
ollama pull phi3                 # Microsoft's small but capable model, ~2.3GB
```

### Hermes Features Not Working with Local Models

**Q: Some Hermes features (like tool calling) don't work?**

A: Not all open-source models support function/tool calling. Models that **do** support it well:

- ✅ Llama 3.1 (all sizes)
- ✅ Qwen 2.5
- ✅ Mistral/Mixtral
- ⚠️ CodeLlama (limited)
- ❌ Older models (Llama 2, etc.)

## Local vs Cloud: When to Use Which

| Scenario | Local (Ollama) | Cloud API |
|----------|:-:|:-:|
| **Privacy-sensitive data** | ✅ Best choice | ⚠️ Data leaves your machine |
| **No internet available** | ✅ Works offline | ❌ Requires connection |
| **Complex reasoning tasks** | ⚠️ Limited by model size | ✅ Access to GPT-4, Claude |
| **High-volume automation** | ✅ No per-token costs | 💰 Gets expensive fast |
| **Quick prototyping** | ✅ Instant, no signup | ✅ Also quick |
| **Enterprise/team use** | ⚠️ Each machine needs setup | ✅ Centralized management |

> **Pro tip:** Many users run both — Ollama for daily tasks and private data, with a cloud API fallback for complex tasks that need GPT-4 class reasoning.
