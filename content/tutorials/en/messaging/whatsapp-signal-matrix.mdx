## What You'll Build

By the end of this tutorial, you'll have Hermes Agent responding to messages on **WhatsApp**, **Signal**, and/or **Matrix** — turning your favorite secure messaging app into a 24/7 AI assistant.

> **Already set up Telegram or Discord?** This guide covers the *other* major platforms. If you haven't set up Telegram yet, start with our [Telegram bot setup guide](/en/tutorials/messaging/telegram-bot-setup) — it's the easiest integration to start with.

## Platform Comparison: Which Should You Choose?

| Feature | WhatsApp | Signal | Matrix |
|---------|:---:|:---:|:---:|
| **User base** | 2.7B+ users | 100M+ users | 80M+ users |
| **E2E Encryption** | ✅ | ✅ (strongest) | ✅ (optional) |
| **API availability** | Business API (paid) | signal-cli (free) | Fully open (free) |
| **Setup difficulty** | 🟡 Medium | 🟡 Medium | 🔴 Advanced |
| **Best for** | Business/customer-facing | Privacy-first users | Self-hosted teams |
| **Rate limits** | Strict (business rules) | Relaxed | None (self-hosted) |
| **Cost** | ~$0.005-0.08/message | Free | Free (self-hosted) |

**Our recommendation:**
- **WhatsApp** if you need to reach the most people (customers, family, colleagues)
- **Signal** if privacy is your top priority
- **Matrix** if you want full control and self-hosting

## Option 1: WhatsApp Integration

WhatsApp integration uses the **WhatsApp Business API** (via Meta's Cloud API). This is the official, approved way to build WhatsApp bots.

### Prerequisites

- A Meta Business account ([business.facebook.com](https://business.facebook.com))
- A phone number dedicated to WhatsApp Business (cannot be your personal number)
- Hermes Agent running on a server with a public URL (or use ngrok for testing)

### Step 1: Set Up WhatsApp Business API

1. Go to [developers.facebook.com](https://developers.facebook.com)
2. Create a new App → Select "Business" type
3. Add the "WhatsApp" product
4. In WhatsApp → API Setup, note your:
   - **Phone Number ID**
   - **WhatsApp Business Account ID**
   - **Temporary Access Token** (generate a permanent one for production)

### Step 2: Configure Hermes WhatsApp Connector

Create a connector config file:

```yaml
# ~/.hermes/connectors/whatsapp.yaml
connector:
  type: whatsapp
  phone_number_id: "YOUR_PHONE_NUMBER_ID"
  business_account_id: "YOUR_BUSINESS_ACCOUNT_ID"
  access_token: "YOUR_ACCESS_TOKEN"
  verify_token: "my-custom-verify-token"  # You choose this
  webhook_port: 3001

settings:
  # WhatsApp-specific settings
  reply_to_groups: false          # Set true to respond in group chats
  max_message_length: 4096       # WhatsApp's limit
  typing_indicator: true          # Show "typing..." before responding
  read_receipts: true             # Mark messages as read
```

### Step 3: Set Up the Webhook

Hermes needs a public URL for WhatsApp to send messages to. For testing, use ngrok:

```bash
# Install ngrok
npm install -g ngrok

# Start the tunnel
ngrok http 3001
```

Copy the HTTPS URL (e.g., `https://abc123.ngrok.io`) and configure it in the Meta Developer Portal:

1. WhatsApp → Configuration → Webhook
2. Callback URL: `https://abc123.ngrok.io/webhook/whatsapp`
3. Verify Token: `my-custom-verify-token` (same as in your config)
4. Subscribe to: `messages`

### Step 4: Start the Connector

```bash
hermes connect whatsapp
```

Expected output:

```
✅ WhatsApp connector active
📱 Listening for messages on +1-XXX-XXX-XXXX
🔗 Webhook endpoint: http://localhost:3001/webhook/whatsapp
```

Now send a message to your WhatsApp Business number — Hermes will respond!

### WhatsApp Limitations to Know

- **24-hour window:** You can only reply within 24 hours of the user's last message (WhatsApp policy)
- **Template messages:** To initiate conversations, you need pre-approved message templates
- **Rate limits:** New numbers start with low limits (~250 messages/day), increasing over time
- **Cost:** Meta charges per conversation ($0.005-0.08 depending on region and type)

## Option 2: Signal Integration

Signal integration uses **signal-cli**, an open-source command-line client for Signal. This is the most privacy-focused option.

### Prerequisites

- A spare phone number for Signal registration (can be a VoIP number)
- Java 17+ installed
- Hermes Agent installed and running

### Step 1: Install signal-cli

```bash
# Download the latest release
SIGNAL_CLI_VERSION="0.13.5"
wget "https://github.com/AsamK/signal-cli/releases/download/v${SIGNAL_CLI_VERSION}/signal-cli-${SIGNAL_CLI_VERSION}.tar.gz"
tar xf "signal-cli-${SIGNAL_CLI_VERSION}.tar.gz"
sudo mv signal-cli-${SIGNAL_CLI_VERSION} /opt/signal-cli
sudo ln -s /opt/signal-cli/bin/signal-cli /usr/local/bin/signal-cli
```

Verify:

```bash
signal-cli --version
# Expected: signal-cli 0.13.5
```

### Step 2: Register Your Signal Number

```bash
# Register (you'll receive an SMS verification code)
signal-cli -u +1YOUR_PHONE_NUMBER register

# Verify with the code you received
signal-cli -u +1YOUR_PHONE_NUMBER verify CODE_FROM_SMS
```

### Step 3: Configure Hermes Signal Connector

```yaml
# ~/.hermes/connectors/signal.yaml
connector:
  type: signal
  phone_number: "+1YOUR_PHONE_NUMBER"
  signal_cli_path: "/usr/local/bin/signal-cli"

settings:
  reply_to_groups: true           # Respond in Signal groups
  allowed_numbers:                # Whitelist (leave empty to allow all)
    - "+1TRUSTED_NUMBER_1"
    - "+1TRUSTED_NUMBER_2"
  max_message_length: 6000
  send_read_receipts: true
```

### Step 4: Start the Signal Connector

```bash
hermes connect signal
```

Expected output:

```
✅ Signal connector active
📱 Listening on +1-XXX-XXX-XXXX
🔒 End-to-end encrypted
```

### Running signal-cli as a Daemon

For 24/7 operation, run signal-cli in daemon mode:

```bash
# Start signal-cli daemon
signal-cli -u +1YOUR_PHONE_NUMBER daemon --socket /tmp/signal-cli.sock &

# Update Hermes config to use the socket
# In signal.yaml:
# signal_cli_mode: socket
# signal_cli_socket: /tmp/signal-cli.sock
```

Or create a systemd service:

```ini
# /etc/systemd/system/signal-cli.service
[Unit]
Description=Signal CLI Daemon
After=network.target

[Service]
ExecStart=/usr/local/bin/signal-cli -u +1YOUR_NUMBER daemon --socket /tmp/signal-cli.sock
Restart=always
User=hermes

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable signal-cli
sudo systemctl start signal-cli
```

## Option 3: Matrix Integration

Matrix is a decentralized, open-source messaging protocol. If you value self-hosting and full control, this is the most powerful option.

### Prerequisites

- A Matrix homeserver (self-hosted Synapse or a public server like matrix.org)
- A dedicated Matrix account for your bot
- Node.js 18+ and Hermes Agent installed

### Step 1: Create a Bot Account

On your Matrix homeserver (or matrix.org):

1. Register a new account (e.g., `@hermes-bot:matrix.org`)
2. Note the access token (Settings → Help & About → Access Token)

Or via API:

```bash
curl -X POST "https://matrix.org/_matrix/client/r0/register" \
  -H "Content-Type: application/json" \
  -d '{"username": "hermes-bot", "password": "SECURE_PASSWORD", "auth": {"type": "m.login.dummy"}}'
```

### Step 2: Configure Hermes Matrix Connector

```yaml
# ~/.hermes/connectors/matrix.yaml
connector:
  type: matrix
  homeserver: "https://matrix.org"       # Or your self-hosted server
  user_id: "@hermes-bot:matrix.org"
  access_token: "YOUR_ACCESS_TOKEN"

settings:
  auto_join: true                         # Auto-join when invited
  reply_to_threads: true                  # Respond in threads
  encryption: true                        # Enable E2E encryption
  allowed_rooms:                          # Whitelist (empty = all rooms)
    - "!room_id_1:matrix.org"
  command_prefix: "!hermes"               # Only respond to messages starting with this
```

### Step 3: Start the Matrix Connector

```bash
hermes connect matrix
```

Expected output:

```
✅ Matrix connector active
🏠 Connected to https://matrix.org
👤 Logged in as @hermes-bot:matrix.org
🔐 Encryption: enabled
📢 Listening in 0 rooms (auto-join enabled)
```

Now invite `@hermes-bot:matrix.org` to any room, and it will auto-join and start responding.

### Self-Hosted Synapse Setup

For maximum privacy, run your own Matrix server:

```bash
# Using Docker
docker run -d \
  --name synapse \
  -v synapse-data:/data \
  -e SYNAPSE_SERVER_NAME=your-domain.com \
  -e SYNAPSE_REPORT_STATS=no \
  -p 8008:8008 \
  matrixdotorg/synapse:latest
```

Then point your Hermes config to `http://localhost:8008`.

## Running Multiple Platforms Simultaneously

The real power comes from connecting Hermes to multiple platforms at once:

```bash
# Start all connectors
hermes connect whatsapp signal matrix
```

Or in your main config:

```yaml
# ~/.hermes/config.yaml
connectors:
  - whatsapp
  - signal
  - matrix
  - telegram    # If you also have Telegram set up
```

Hermes maintains a **unified conversation context** across all platforms. A conversation started on Signal can be continued on WhatsApp — Hermes remembers everything.

## Troubleshooting

### WhatsApp Webhook Not Receiving Messages

**Q: Set up everything but messages aren't coming through?**

1. Check webhook verification: `curl https://your-url/webhook/whatsapp`
2. Verify the callback URL is HTTPS (WhatsApp requires it)
3. Check Meta Developer Portal for error logs
4. Ensure your app is in "Live" mode, not "Development"

### Signal Registration Failed

**Q: Can't register the phone number?**

A: Signal has anti-bot measures. Try:

1. Use a real phone number (not all VoIP numbers work)
2. Wait 24 hours and retry if rate-limited
3. Try the CAPTCHA-based registration: `signal-cli -u +1NUMBER register --captcha CAPTCHA_TOKEN`

### Matrix Encryption Errors

**Q: Getting "Unable to decrypt message" errors?**

A: Matrix encryption requires device verification:

```bash
# Verify all sessions
hermes matrix verify-all
```

Or disable encryption in the config if you don't need it.

## Security Best Practices

When exposing Hermes to messaging platforms, security is critical:

1. **Whitelist trusted contacts** — Don't let strangers interact with your agent
2. **Set rate limits** — Prevent abuse with message-per-minute caps
3. **Use separate API keys** — Each connector should have its own credentials
4. **Monitor logs** — Check `~/.hermes/logs/` for suspicious activity
5. **Keep signal-cli updated** — Security patches are frequent
