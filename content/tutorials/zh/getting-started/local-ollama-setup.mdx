## 这篇教程能帮你做什么？

读完这篇文章，你会拥有一个**完全运行在自己电脑上的 AI Agent**——不需要买 API、不需要翻墙、不需要注册任何账号，断网状态下也能用。

> 适合你的情况：
> - 💰 不想为 API 付费（GPT-4 一个月轻松烧掉几十美元）
> - 🔒 处理敏感数据，不希望对话内容上传到云端
> - 🌐 网络环境不稳定，经常连不上国外 API
> - 🧪 想折腾开源模型，自由切换尝试

## 前置条件

- **已安装 Hermes Agent** — 还没装？先看 [安装指南](/zh/tutorials/getting-started/installation)
- **硬件要求：**

| 模型大小 | 最低内存 | 推荐配置 | 体验 |
|---------|---------|---------|------|
| 7B 模型 | 8GB RAM | 16GB RAM | 流畅，每秒 10-20 字 |
| 13B 模型 | 16GB RAM | 32GB RAM | 流畅，质量明显提升 |
| 70B 模型 | 48GB+ RAM | 64GB + GPU | 接近 GPT-4 水平 |

> **没有独立显卡也能跑！** Ollama 支持纯 CPU 运行，只是速度慢一些。有 NVIDIA 显卡（6GB 显存以上）会快 3-5 倍。苹果 M 系列芯片表现尤其出色。

## 第 1 步：安装 Ollama

Ollama 可以理解为「本地模型的 Docker」——一条命令就能下载和运行各种 AI 模型。

### Windows 用户（最常见）

1. 下载安装包：访问 [ollama.com/download](https://ollama.com/download)
2. 双击安装，一路 Next
3. 安装完成后，Ollama 会作为后台服务自动运行

打开 PowerShell 验证：

```powershell
ollama --version
# 预期输出: ollama version 0.5.x
```

> **Windows 注意事项：** Ollama 在 Windows 上原生支持，不需要 WSL2。但如果你之前在 WSL2 里安装了 Hermes，那 Ollama 也需要装在 WSL2 里（见下方 Linux 步骤）。

### macOS / Linux / WSL2

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

如果下载速度慢，可以设置代理：

```bash
export https_proxy=http://127.0.0.1:7890
curl -fsSL https://ollama.com/install.sh | sh
```

### 验证安装

```bash
curl http://localhost:11434
# 预期输出: Ollama is running
```

看到这个输出，说明 Ollama 的本地 API 服务器已经启动，运行在 `http://localhost:11434`。

## 第 2 步：选择并下载模型

这是最关键的一步。**模型选错，体验天差地别。** 以下是我们实测后的推荐：

### 🇨🇳 国产模型推荐（中文场景首选）

| 模型 | 大小 | 内存 | 中文能力 | 推荐场景 | 下载命令 |
|------|------|------|:---:|---------|---------|
| **通义千问 Qwen 2.5 7B** | 4.4GB | 8GB | ⭐⭐⭐⭐⭐ | 中文对话、写作、翻译 | `ollama pull qwen2.5` |
| **通义千问 Qwen 2.5 14B** | 8.9GB | 16GB | ⭐⭐⭐⭐⭐ | 复杂推理、长文分析 | `ollama pull qwen2.5:14b` |
| **DeepSeek Coder V2** | 8.9GB | 16GB | ⭐⭐⭐⭐ | 写代码、代码审查 | `ollama pull deepseek-coder-v2` |
| **DeepSeek V2.5** | 8.9GB | 16GB | ⭐⭐⭐⭐⭐ | 全能选手 | `ollama pull deepseek-v2.5` |

### 🌍 国际模型推荐

| 模型 | 大小 | 内存 | 英文能力 | 推荐场景 | 下载命令 |
|------|------|------|:---:|---------|---------|
| **Llama 3.1 8B** | 4.7GB | 8GB | ⭐⭐⭐⭐⭐ | 通用任务 | `ollama pull llama3.1` |
| **Mistral 7B** | 4.1GB | 8GB | ⭐⭐⭐⭐ | 速度快、指令跟随好 | `ollama pull mistral` |
| **Phi-3 Mini** | 2.3GB | 4GB | ⭐⭐⭐ | 超轻量、低配电脑 | `ollama pull phi3` |

### 我们的推荐

**如果你主要用中文：**

```bash
# 首选！中文能力最强的 7B 模型
ollama pull qwen2.5
```

**如果你电脑配置高（16GB+ 内存）：**

```bash
# 14B 模型，中文理解力大幅提升
ollama pull qwen2.5:14b
```

**如果你主要写代码：**

```bash
# 代码能力最强的开源模型
ollama pull deepseek-coder-v2
```

下载完成后，确认模型列表：

```bash
ollama list
```

预期输出：

```
NAME              ID           SIZE    MODIFIED
qwen2.5:latest    f1e2d3c4b5   4.4 GB  刚刚
```

### 快速测试模型

先单独测试一下模型能不能正常工作：

```bash
ollama run qwen2.5 "用一句话介绍你自己"
```

预期输出类似：

```
我是通义千问，一个由阿里云开发的大型语言模型，可以帮你回答问题、写文章、写代码等。
```

按 `Ctrl+D` 退出 Ollama 对话。

## 第 3 步：连接 Hermes Agent 到 Ollama

### 方式 A：环境变量（快速上手）

```bash
export OPENAI_API_BASE=http://localhost:11434/v1
export OPENAI_API_KEY=ollama    # Ollama 不需要真实密钥，但 Hermes 需要这个字段
export DEFAULT_MODEL=qwen2.5
```

Windows PowerShell 用户：

```powershell
$env:OPENAI_API_BASE = "http://localhost:11434/v1"
$env:OPENAI_API_KEY = "ollama"
$env:DEFAULT_MODEL = "qwen2.5"
```

然后启动 Hermes：

```bash
hermes
```

### 方式 B：配置文件（一劳永逸）

编辑 Hermes 配置文件 `~/.hermes/config.yaml`：

```yaml
provider:
  type: openai-compatible
  base_url: http://localhost:11434/v1
  api_key: ollama    # 占位符——Ollama 会忽略这个值
  model: qwen2.5

# 针对本地模型的优化参数
settings:
  max_tokens: 2048        # 7B 模型建议不超过 2048
  temperature: 0.7
  context_window: 8192    # 保持合理的上下文窗口
```

### 验证连接

```bash
hermes "你好！请告诉我你是什么模型？"
```

你应该看到类似这样的回复：

```
🤖 你好！我是通义千问（Qwen 2.5），通过 Ollama 在你本地运行。
   有什么我可以帮你的吗？
```

🎉 **恭喜！你的 AI Agent 现在完全运行在本地了！** 所有对话数据都在你自己的电脑上，没有任何数据上传到云端。

## 第 4 步：性能优化

### 配置太低？试试这些方法

**方法 1：使用量化版本（最有效）**

量化会压缩模型大小，牺牲少量精度换取大幅性能提升：

```bash
# 4-bit 量化版，大小减半，速度翻倍
ollama pull qwen2.5:7b-q4_0

# 超低配方案：2GB 内存就能跑
ollama pull phi3:mini
```

**方法 2：调整 Hermes 参数**

```yaml
# ~/.hermes/config.yaml
settings:
  max_tokens: 1024       # 缩短回复 = 更快
  context_window: 2048   # 减小上下文 = 省内存
```

**方法 3：关闭不需要的后台程序**

本地模型会吃掉大量内存。关掉 Chrome（吃内存大户）、Docker 等不需要的程序。

### NVIDIA 显卡用户

Ollama 会自动检测 NVIDIA 显卡。确认是否在用 GPU：

```bash
ollama ps
```

如果看到 `100% GPU`，说明模型完全加载到显卡了：

```
NAME      ID          SIZE     PROCESSOR      UNTIL
qwen2.5   f1e2d3c4    6.1 GB   100% GPU       4 minutes from now
```

> **如果显示 0% GPU：** 检查是否安装了最新的 NVIDIA 驱动和 CUDA。运行 `nvidia-smi` 确认。

### 苹果 M 系列芯片

好消息——Ollama 对 Apple Silicon 做了深度优化，利用统一内存架构。16GB 内存的 MacBook 可以流畅运行 7B 模型，速度达到 30+ tokens/sec，体验非常好。

## 多模型切换：不同任务用不同模型

Ollama 支持同时下载多个模型，随时切换：

```bash
# 下载中文通用模型
ollama pull qwen2.5

# 下载代码专用模型
ollama pull deepseek-coder-v2

# 聊天用通义千问
hermes --model qwen2.5 "帮我写一封请假邮件"

# 写代码用 DeepSeek
hermes --model deepseek-coder-v2 "用 Python 写一个爬虫，抓取豆瓣电影 Top250"
```

> **技巧：** 在配置文件中设置默认模型，命令行中用 `--model` 覆盖，两全其美。

## 常见问题排查

### Ollama 没有运行

**Q: Hermes 报错 "Connection refused" 或连接不上？**

```bash
# 手动启动 Ollama
ollama serve

# Linux 系统：
sudo systemctl start ollama
sudo systemctl status ollama
```

### 速度太慢

**Q: 一秒只出 2-3 个字，像打字机一样？**

A: 模型太大了，你的硬件跑不动。解决方案：

1. 换小模型：`ollama pull phi3`（2.3GB，飞快）
2. 用量化版：`ollama pull qwen2.5:7b-q4_0`
3. 减小上下文窗口到 `2048`

### 内存不够

**Q: 报错 "out of memory"？**

A: 模型放不进内存。试试：

```bash
# 超轻量模型，4GB 内存就能跑
ollama pull phi3:mini

# 或者用更激进的量化
ollama pull qwen2.5:7b-q2_K
```

### 下载模型太慢

**Q: 下载速度只有几十 KB/s？**

A: 设置代理加速：

```bash
# HTTP 代理
export https_proxy=http://127.0.0.1:7890
ollama pull qwen2.5

# 或者用国内镜像（如果有的话）
# 注意：截至 2026 年 4 月，Ollama 暂无官方国内镜像
# 建议使用稳定的代理工具
```

### 工具调用不工作

**Q: Hermes 的技能系统和工具调用用不了？**

A: 不是所有模型都支持 function calling。以下是兼容性列表：

| 模型 | 工具调用支持 |
|------|:---:|
| Qwen 2.5 | ✅ 完整支持 |
| Llama 3.1 | ✅ 完整支持 |
| DeepSeek V2.5 | ✅ 完整支持 |
| Mistral | ✅ 支持 |
| DeepSeek Coder V2 | ⚠️ 部分支持 |
| Phi-3 | ⚠️ 有限支持 |

**推荐：** 如果你需要用 Hermes 的技能系统，选 **Qwen 2.5** 或 **Llama 3.1**。

## 本地 vs 云端：什么时候用哪个？

| 场景 | 本地（Ollama） | 云端 API |
|------|:---:|:---:|
| **处理公司/个人敏感数据** | ✅ 数据不出本机 | ⚠️ 数据上传到第三方 |
| **没有网络** | ✅ 离线可用 | ❌ 必须联网 |
| **需要顶级推理能力** | ⚠️ 受模型大小限制 | ✅ GPT-4、Claude 级别 |
| **大量日常任务** | ✅ 零边际成本 | 💰 token 费用累积快 |
| **低配电脑** | ⚠️ 至少 8GB 内存 | ✅ 云端算力不受限 |

> **我们的建议：** 日常任务用本地 Ollama（省钱 + 隐私），遇到复杂任务时切换到云端 API。Hermes 支持随时切换，两者可以共存。
