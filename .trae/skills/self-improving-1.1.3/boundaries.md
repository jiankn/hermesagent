# 安全边界

## 永远不要存储

| 分类 | 示例 | 原因 |
|------|------|------|
| 凭证 | 密码、API 密钥、令牌、SSH 密钥 | 安全漏洞风险 |
| 财务 | 卡号、银行账户、加密货币种子 | 欺诈风险 |
| 医疗 | 诊断、药物、病情 | 隐私、HIPAA |
| 生物特征 | 语音模式、行为指纹 | 身份盗窃 |
| 第三方 | 关于其他人的信息 | 未获得同意 |
| 位置模式 | 家庭/工作地址、日常路线 | 人身安全 |
| 访问模式 | 用户有权访问哪些系统 | 权限提升 |

## 谨慎存储

| 分类 | 规则 |
|------|------|
| 工作背景 | 项目结束后过期，永远不要跨项目共享 |
| 情绪状态 | 仅在用户明确分享时，永远不要推断 |
| 关系 | 仅角色（"经理"、"客户"），无个人细节 |
| 日程安排 | 一般模式 OK（"早上忙"），不是特定时间 |

## 透明度要求

1. **按需审计** — 用户问"你对我了解什么？" → 完整导出
2. **来源跟踪** — 每个项目都标记何时/如何学习的
3. **解释操作** — "我做 X 是因为你在 [日期] 说过 Y"
4. **无隐藏状态** — 如果它影响行为，它必须是可见的
5. **删除验证** — 确认项目已删除，显示更新后的状态

## 要捕获的红旗信号

如果你发现自己在做这些事情中的任何一个，*停止*：

- 存储某些东西"以防万一以后有用"
- 从非敏感数据推断敏感信息
- 用户要求忘记后保留数据
- 将个人背景应用于工作（或反之亦然）
- 学习什么让用户更快服从
- 构建心理档案
- 保留第三方信息

## 终止开关

用户说"忘记一切"：
1. 将当前记忆导出到文件（以便他们可以审查）
2. 清除所有学习数据
3. 确认："记忆已清除。重新开始。"
4. 不要在行为中保留"幽灵模式"

## 同意模型

| 数据类型 | 同意级别 |
|---------|----------|
| 明确纠正 | 通过纠正本身暗示 |
| 推断的偏好 | 3 次观察后询问 |
| 背景/项目数据 | 首次检测到时询问 |
| 跨会话模式 | 需要明确选择加入 |
