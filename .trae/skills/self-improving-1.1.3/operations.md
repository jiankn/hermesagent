# 记忆操作

## 用户命令

| 命令 | 操作 |
|------|------|
| "你对 X 了解什么？" | 搜索所有层级，返回带来源的匹配 |
| "显示我的记忆" | 显示 memory.md 内容 |
| "显示 [项目] 模式" | 加载并显示特定命名空间 |
| "忘记 X" | 从所有层级中删除，确认删除 |
| "忘记一切" | 完全擦除，带导出选项 |
| "最近有什么变化？" | 显示最后 20 个纠正 |
| "导出记忆" | 生成可下载的归档 |
| "记忆状态" | 显示层级大小、上次压缩、健康状况 |

## 自动操作

### 会话开始时
1. 加载 memory.md（热层）
2. 检查 index.md 以获取背景提示
3. 如果检测到项目 → 预加载相关命名空间

### 收到纠正时
```
1. 解析纠正类型（偏好、模式、覆盖）
2. 检查是否重复（存在于任何层级中）
3. 如果是新的：
   - 添加到 corrections.md 并带时间戳
   - 增加纠正计数器
4. 如果是重复的：
   - 提高计数器，更新时间戳
   - 如果计数器 >=3：询问确认为规则
5. 确定命名空间（全局、领域、项目）
6. 写入适当的文件
7. 更新 index.md 行数
```

### 模式匹配时
应用学习的模式时：
```
1. 找到模式来源（文件:行）
2. 应用模式
3. 引用来源："使用 X（来自 memory.md:15）"
4. 记录使用情况以进行过期跟踪
```

### 每周维护（Cron）
```
1. 扫描所有文件以查找过期候选
2. 将 30+ 天未使用的移动到温
3. 将 90+ 天未使用的归档到冷
4. 如果任何文件 > 限制，运行压缩
5. 更新 index.md
6. 生成每周摘要（可选）
```

## 文件格式

### memory.md（热）
```markdown
# 自我改进记忆

## 已确认偏好
- format：项目符号优于散文（已确认 2026-01）
- tone：直接，不模棱两可（已确认 2026-01）

## 活跃模式
- "看起来不错" = 批准继续（使用 15 次）
- 单个表情符号 = 已确认（使用 8 次）

## 最近（过去 7 天）
- MVP 喜欢 SQLite（02-14 纠正）
```

### corrections.md
```markdown
# 纠正日志

## 2026-02-15
- [14:32] 将冗长解释改为项目符号摘要
  类型：communication
  背景：Telegram 回应
  已确认：pending (1/3)

## 2026-02-14
- [09:15] MVP 使用 SQLite 不是 Postgres
  类型：technical
  背景：数据库讨论
  已确认：yes（说"总是"）
```

### projects/{name}.md
```markdown
# 项目：my-app

继承：global、domains/code

## 模式
- 使用 Tailwind（项目标准）
- 不要 Prettier（仅 eslint）
- 通过 GitLab CI 部署

## 覆盖
- 分号：yes（覆盖全局 no-semi）

## 历史
- 创建：2026-01-15
- 上次活跃：2026-02-15
- 纠正：12
```

## 边缘情况处理

### 检测到矛盾
```
模式 A："使用制表符"（全局，已确认）
模式 B："使用空格"（项目，今天纠正）

解决：
1. 项目覆盖全局 → 此项目使用空格
2. 在 corrections.md 中记录冲突
3. 询问："空格应该仅适用于此项目还是所有地方？"
```

### 用户改变主意
```
旧："总是使用正式语气"
新："实际上，随意的就好"

操作：
1. 归档带时间戳的旧模式
2. 将新模式添加为暂定
3. 保留归档以供参考（"你以前更喜欢正式"）
```

### 背景模糊
```
用户说："记住我喜欢 X"

但是哪个命名空间？
1. 检查当前背景（项目？领域？）
2. 如果不清楚，询问："这应该全局应用还是仅在这里？"
3. 默认为最具体的活跃背景
```
