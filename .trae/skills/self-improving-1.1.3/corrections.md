# 纠正日志 — 模板

> 首次使用技能时，此文件会在 `~/self-improving/corrections.md` 中创建。
> 保留最后 50 个纠正。较旧的条目会被评估以提升或归档。

## 示例条目

```markdown
## 2026-02-19

### 14:32 — 代码风格
- **纠正：** "使用 2 空格缩进，不是 4"
- **背景：** 编辑 TypeScript 文件
- **计数：** 1（第一次出现）

### 16:15 — 沟通
- **纠正：** "不要以'好问题！'开头回应"
- **背景：** 聊天回应
- **计数：** 3 → **提升到 memory.md**

## 2026-02-18

### 09:00 — 项目：website
- **纠正：** "对于这个项目，总是使用 Tailwind"
- **背景：** CSS 讨论
- **操作：** 添加到 projects/website.md
```

## 日志格式

每个条目包括：
- **时间戳** — 纠正发生的时间
- **纠正** — 用户说了什么
- **背景** — 是什么触发了它
- **计数** — 多少次（用于提升跟踪）
- **操作** — 存储在哪里（如果提升了）
