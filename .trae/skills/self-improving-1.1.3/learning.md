# 学习机制

## 什么触发学习

| 触发器 | 置信度 | 操作 |
|-------|--------|------|
| "不，改为做 X" | 高 | 立即记录纠正 |
| "我以前告诉过你..." | 高 | 标记为重复，提高优先级 |
| "总是/从不做 X" | 已确认 | 提升为偏好 |
| 用户编辑你的输出 | 中 | 记录为暂定模式 |
| 相同纠正 3 次 | 已确认 | 询问是否永久化 |
| "对于这个项目..." | 限定范围 | 写入项目命名空间 |

## 什么不触发学习

- 沉默（不是确认）
- 任何事情的单一实例
- 假设性讨论
- 第三方偏好（"John 喜欢..."）
- 群聊模式（除非用户确认）
- 隐含的偏好（永远不要推断）

## 纠正分类

### 按类型
| 类型 | 示例 | 命名空间 |
|------|------|---------|
| 格式 | "使用项目符号而不是散文" | global |
| 技术 | "SQLite 不是 Postgres" | domain/code |
| 沟通 | "更短的消息" | global |
| 项目特定 | "这个仓库使用 Tailwind" | projects/{name} |
| 个人特定 | "Marcus 想要 BLUF" | domains/comms |

### 按范围
```
全局：适用于所有地方
  └── 领域：适用于分类（代码、写作、通信）
       └── 项目：适用于特定背景
            └── 临时：仅适用于本次会话
```

## 确认流程

3 次类似纠正后：
```
代理："我注意到你更喜欢 X 而不是 Y（已纠正 3 次）。
        我应该总是这样做吗？
        - 是的，总是
        - 仅在 [背景] 中
        - 不，视情况而定"

用户："是的，总是"

代理：→ 移动到已确认偏好
       → 从纠正计数器中删除
       → 将来使用时引用来源
```

## 模式演变

### 阶段
1. **暂定** — 单次纠正，观察重复情况
2. **涌现** — 2 次纠正，可能的模式
3. **待处理** — 3 次纠正，询问确认
4. **已确认** — 用户批准，永久除非被撤销
5. **已归档** — 90+ 天未使用，保留但不活跃

### 撤销
用户总是可以撤销：
```
用户："实际上，我改变了对 X 的主意"

代理：
1. 归档旧模式（保留历史）
2. 记录带时间戳的撤销
3. 将新偏好添加为暂定
4. "好的。我现在会做 Y。（以前：X，已归档）"
```

## 反模式

### 永远不要学习
- 什么让用户更快服从（操纵）
- 情绪触发因素或漏洞
- 来自其他用户的模式（即使共享设备）
- 任何感觉"令人毛骨悚然"的东西

### 避免
- 从单一实例过度概括
- 学习风格而不是实质
- 假设偏好稳定性
- 忽略背景变化

## 质量信号

### 好的学习
- 用户明确说明偏好
- 跨背景一致的模式
- 纠正确实改善结果
- 用户在被问到时确认

### 坏的学习
- 从沉默中推断
- 与最近的行为矛盾
- 仅在狭窄背景中有效
- 用户从未确认
